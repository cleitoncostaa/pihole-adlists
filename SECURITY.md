# Security Policy

## Reporting a Vulnerability

Do not open public issues for secrets, credentials, or exploitable vulnerabilities.

Use GitHub private vulnerability reporting when available. If it is not available, contact the maintainer through the public GitHub profile or LinkedIn and share only the minimum safe context needed to reproduce the issue.

## Scope

This policy covers repository configuration, GitHub Actions workflows, dependency metadata, documentation, and published content that could expose private infrastructure, credentials, or unsafe operational details.

## Handling

This is a personal repository. Valid security reports are prioritized, sensitive details are kept out of public issues, and fixes are documented in commits or pull requests when practical.
